# TASK-E2E-RELIABILITY-001 User Manual

## 1. 사용자 영향

화면 구성, 문구와 구매 업무 정책은 바뀌지 않는다. 구매정보 수정 화면이 여러 번 로드되는 개발·느린 네트워크 조건에서도 사용자가 추가한 행이 늦은 응답 때문에 사라지지 않도록 안정화한다.

## 2. 확인 방법

1. 구매 권한 사용자로 프로젝트의 구매 tab을 연다.
2. `구매정보 수정`을 선택한다.
3. `행 추가`를 한 번 누른다.
4. 빈 행이 정확히 한 개 추가되는지 확인한다.
5. 통상납기와 발주일 등 값을 입력한다.
6. 잠시 기다려도 입력 행과 값이 사라지지 않는지 확인한다.
7. 저장 후 구매정보 조회에 값이 표시되는지 확인한다.

## 3. 문의가 필요한 경우

- 행 추가 후 빈 행이 사라짐
- 버튼 한 번에 행이 둘 이상 추가됨
- 입력 도중 기존 값으로 되돌아감
- 저장하지 않았는데 화면이 상세 page로 이동함

## 4. 변경되지 않은 사항

- 구매정보 필드와 권한
- 저장 API와 DB schema
- Excel 업로드와 자재 입고 정책
- 모바일 카드와 desktop table 구성

- [x] 입력값이 늦은 load로 사라지지 않음
- [x] 저장과 상세 조회 정상
- [x] 기존 권한과 모바일 표시 정상
