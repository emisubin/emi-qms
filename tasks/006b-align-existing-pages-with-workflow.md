# TASK-006B 기존 페이지 18단계 프로세스 연결 보강

## 목적

TASK-006B는 TASK-006A에서 추가된 workflow, 내 업무, 알림 기반 구조를 기존 프로젝트/설계/생산관리/구매/자재 화면과 맞추는 작업이다.

새 공통 모듈을 만들지 않고, 이미 개발된 페이지의 상태, 용어, 완료 기준, 링크, Excel header를 최종 18단계 업무 흐름에 맞춰 정렬한다.

## 구현 범위

- 프로젝트 목록/상세의 상태와 진행률을 workflow 기준으로 통합한다.
- 생산관리 단계 완료 기준을 확장 담당자 구조 기준으로 정리한다.
- 프로젝트 생성/수정/상세에 FAT 필요 여부를 추가한다.
- 설계/패널명·사이즈 단계가 기존 패널정보 화면과 연결되도록 사용자 표시 용어를 정리한다.
- 구매정보에 업체 필드를 추가한다.
- 구매 workflow 완료 판정이 필수 구매 항목과 실제 입력 여부를 반영하는지 유지/보강한다.
- 내 업무/알림 링크가 구현된 stage section으로 이동하도록 정리한다.
- 자재/출하 관련 사용자 표시 용어를 최종 흐름과 충돌하지 않게 정리한다.
- 프로젝트/구매 Excel template/header 용어를 정리한다.
- 프로젝트 상세 section과 workflow 단계를 연결한다.

## 제외 범위

- Pending List 공통 모듈
- 품질 부적합, 고객사 PUNCH LIST, 제조 중단
- IQC/LQC/OQC/FAT 체크리스트
- 검사성적서 PDF 출력
- 제조 체크리스트와 작업 시작/종료
- 실제 자재 도착/IQC 요청/입고 확정 분리 기능
- 키팅 완료 실제 기능
- 포장 완료/출발 처리/납품 완료 실제 물류 기능
- 세금계산서/영업 정산 화면
- Microsoft 365 로그인
- 관리자 기준정보 전체 페이지
- 모든 페이지 Excel export 공통 기능
- QR 이미지 생성/출력
- 기존 QR 생성 가능 조건 변경

| 페이지 | 수정 방향 |
| --- | --- |
| 프로젝트 목록 | workflow 기준 상태/진행률을 사용한다. |
| 프로젝트 생성 | FAT 필요 여부를 추가한다. 기본값은 아니오다. |

## Workflow 상태/진행률 통합 방향

- 프로젝트 진행률은 완료된 필수 workflow 단계 수 / 전체 필수 workflow 단계 수로 계산한다.
- FAT 필요 프로젝트는 FAT 단계를 denominator에 포함한다.
- FAT 불필요 프로젝트는 FAT 단계를 선택 제외 상태로 처리하고 denominator에서 제외한다.
- 프로젝트 관리 상태 `Cancelled`, `OnHold`, `Completed`는 workflow 현재 단계보다 우선한다.
- 같은 프로젝트의 목록/상세/API 진행률이 서로 다르면 안 된다.

## FAT 필요 여부

- 프로젝트 생성/수정에서 `FAT 필요 여부`를 선택한다.
- 기본값은 아니오다.
- 기존 프로젝트는 false로 처리한다.
- FAT 필요 여부가 true일 때만 workflow 14단계를 필수 진행률에 포함한다.

## 구매 업체 헤더

- 구매정보에 `업체` 필드를 추가한다.
- 필수값은 아니다.
- 구매 조회, 수정, 프로젝트 상세 구매 section, Excel template/preview/apply에 반영한다.
- 업체 master 관리는 후속 TASK로 남긴다.

## 설계/패널정보 용어 정리

- 내부 API/코드명 `PanelInformation`은 유지할 수 있다.
- 사용자 화면에서는 `설계`, `패널명·사이즈 수정`, `설계 정보 완료` 기준으로 표시한다.
- QR 가능 조건은 변경하지 않는다.

## 내 업무/알림 Deep Link

- 구현된 화면은 프로젝트 상세 section으로 이동한다.
- 아직 구현되지 않은 화면은 프로젝트 상세 기본 화면으로 이동한다.
- 404 링크는 만들지 않는다.

| Stage | Link |
| --- | --- |
| ProductionPlanning | `/projects/{id}?section=production-planning` |
| DesignPanelInfo | `/projects/{id}?section=panels` |
| ProcurementInfo | `/projects/{id}?section=procurement` |
| ReceiptConfirmed | `/materials/receipts` |
| 그 외 미구현 단계 | `/projects/{id}` |

## 자재/출하 용어 정리

- 이번 TASK에서 실제 자재 도착/IQC 요청/입고 확정/키팅 기능을 분리하지 않는다.
- 현재 자재 기능은 자재 입고 처리 또는 입고 확정 성격으로 유지한다.
- 사용자 화면에서 `출하 완료`는 `납품 완료`로 정리한다.

## 테스트 계획

- Backend: 프로젝트 FAT 생성/수정/상세, workflow 진행률, 생산관리 완료 기준, 구매 업체 저장/조회/Excel, deep link mapping.
- Frontend: FAT UI, 설계/패널명·사이즈 용어, 구매 업체 field, 내 업무/알림 section 이동, 납품 완료 용어.
- Full-Stack E2E: FAT field, 설계 용어, 구매 업체 저장, workflow 진행률, deep link, 자재 용어 smoke.
- Migration: 0014 신규/기존 DB 적용.
