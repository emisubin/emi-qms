# 10. 핵심 테스트 시나리오

## 시나리오 1: 정상 흐름

영업 프로젝트 등록 → 제품 생성 → 설계 출도 → 생산계획 → 제조 완료 → 검사 적합 → 품질 승인 → 포장 → 출발 → 납품 증빙 → 영업 확인 → 종료.

## 시나리오 2: 부적합과 재검사

품질 부적합 등록 → NCR 생성 → 출하 잠금 → 제조 조치·사진 → 재검사 적합 → NCR 종료 → 품질 승인.

검증: 최초 검사결과가 덮어써지지 않아야 합니다.

## 시나리오 3: 재검사 재부적합

재검사에서 다시 부적합 → 기존 NCR 재조치 또는 정책에 따른 신규 이력 → 제조 재알림 → 품질 승인 불가.

## 시나리오 4: 권한 없는 접근

프로젝트 A 권한만 있는 사용자가 프로젝트 B의 제품 ID 또는 URL을 요청 → 서버 거부 → 업무 데이터 미노출 → 감사 로그 기록.

## 시나리오 5: 중복 제출

모바일 네트워크 지연 중 완료 버튼을 두 번 누름 → 단일 완료 이력만 생성 → 사용자에게 현재 결과 표시.

## 시나리오 6: 승인 후 정정

품질 승인 이후 측정값 정정 요청 → 사유와 승인 필요 → 원본과 변경값 모두 보존 → PDF 재발행 정책 적용.

## 시나리오 7: 분할 출하

프로젝트 내 일부 제품만 출발·납품 완료 → 해당 제품만 상태 변경 → 프로젝트는 일부 출하로 계산.
