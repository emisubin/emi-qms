# 8. 확정이 필요한 의사결정

아래 항목은 실제 개발 전에 부서 합의가 필요합니다.

| 번호 | 결정 항목 | 현재 제안 |
|---|---|---|
| D-001 | 프로젝트번호 원천 | 영업 등록 또는 기존 PJT 관리 데이터 연동 |
| D-002 | 제품번호 규칙 | `{프로젝트번호}-P{순번 2자리}` |
| D-003 | QR 부착 시점 | 생산계획 확정 후 외함 확인 시점 |
| D-004 | 설계 미출도 긴급생산 | 책임자 승인과 사유가 있는 경우만 허용 |
| D-005 | 품질 승인자 | 검사자와 분리 여부를 제품군별 확정 |
| D-006 | 조건부 승인·특채 | MVP 포함 여부와 승인권자 |
| D-007 | 납품 완료 등록자 | 물류 또는 기사/외부 사용자 여부 |
| D-008 | 거래명세서 형식 | 스캔 PDF, 사진, 전자서명 중 허용 범위 |
| D-009 | 사진 보관기간 | 고객·품질 요구에 따라 확정 |
| D-010 | Teams 알림 방식 | Power Automate 또는 Graph API |
| D-011 | 고객사별 검사양식 | MVP 대상 제품군 선정 |
| D-012 | 모바일 공용기기 | 개인 스마트폰/회사 태블릿 정책 |

결정 시 날짜, 결정자, 근거를 기록하고 필요하면 ADR로 승격합니다.
