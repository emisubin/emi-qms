# 보안 정책

## 민감정보

운영 비밀번호, API 키, 인증서, 토큰, 실고객 데이터는 저장소에 커밋하지 않습니다. 발견 즉시 해당 비밀을 폐기·재발급하고 Git 이력 노출 범위를 점검합니다.

## 취약점 보고

보안 문제는 공개 Issue에 올리지 않고 사내 지정 담당자에게 비공개로 보고합니다. 보고에는 재현 조건, 영향 범위, 관련 버전, 임시 완화책을 포함합니다.

## 운영 배포 전 필수 검증

- 프로젝트·제품별 서버 권한 검증
- 승인 데이터 정정 통제
- 파일 업로드·다운로드 권한 및 형식 검증
- 로그인·권한 변경·대량 다운로드 감사 로그
- 종속성 취약점 검사
- 백업 복구 시험
- 시험 서버에서의 승인된 보안 점검
