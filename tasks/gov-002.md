## 2. 목적

Current checkout에서 제거된 실제 사용자 이름이 public Git history에 남은 P2를 조사하고, risk acceptance가 아니라 coordinated history rewrite로 해소하기로 결정한다.

이 Task는 정책 결정을 고정한다. Rewrite, force push, visibility 변경과 clone 재동기화는 별도 `TASK-GOV-HISTORY-REWRITE-001`의 계획·승인·실행 범위다.

## 4. 승인된 정책

`COORDINATED_HISTORY_REWRITE`를 선택한다.

- Main만이 아니라 개인정보 history를 포함하는 모든 published ref를 같은 maintenance에서 처리한다.
- Current checkout의 익명 표현을 replacement 기준으로 사용한다.
- Rewrite는 fresh task-owned mirror에서 수행한다.
- Replacement mapping과 pre-rewrite backup은 Repository 밖 승인된 보안 위치에 둔다.
- Force push, visibility 변경, secure backup과 contributor re-clone은 개별 승인 경계를 유지한다.
- 이미 존재할 수 있는 외부 clone·download를 회수했다고 표현하지 않는다.

## 5. Risk owner와 책임

Risk owner는 개인명이 아닌 역할 `Repository owner / security owner`다.

- Rewrite 실행 승인과 maintenance 시간을 확정한다.
- Public 유지 또는 temporary private containment를 결정한다.
- Secure backup의 보존 기간과 삭제를 승인한다.
- Contributor·automation re-clone 완료를 확인한다.
- Old history 재유입 또는 GitHub cache 잔존 위험을 수용하거나 추가 조치를 승인한다.
