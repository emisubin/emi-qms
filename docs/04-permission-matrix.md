# 4. 역할·권한표

권한은 역할뿐 아니라 소속, 담당 프로젝트, 제품 범위, 현재 상태를 함께 확인합니다.

| 역할 | 주요 조회 | 주요 입력·수정 | 승인·완료 |
|---|---|---|---|
| 영업 | 담당 프로젝트 전체 진행, 출하·납품 증빙 | 프로젝트, 납기, 포장요구, 특이사항 | 납품 증빙 확인, 제품 종료 |
| 설계 | 프로젝트·제품·생산계획 | 도면/BOM Revision, 출도, 변경정보 | 설계 출도 |
| 구매 | 프로젝트, 자재·생산 일정 | 입고목표, 실입고, 미입고 사유 | 입고계획 확정 |
| 생산관리 | 전체 프로젝트·제품 상태 | 생산계획, 담당팀, 우선순위, QR 관리 | 예외 생산 승인 후보 |
| 제조 | 담당 제품, 도면 Revision, 품질 부적합 | 제조단계, 작업사진, NCR 조치 | 제조 완료, 조치 완료 |
| 품질 검사자 | 생산·검사·조치 정보 | 검사결과, 측정값, 사진, NCR, 재검사 | 검사 완료 |
| 품질 승인자 | 모든 품질이력 | 승인 전 정정 요청 검토 | 품질 최종 승인 |
| 물류 | 품질 승인 제품, 포장요구 | 포장, 박스, 배차, 출발, 납품증빙 | 출발·납품 완료 |
| 조회전용 | 허용된 프로젝트 | 없음 | 없음 |
| 시스템 관리자 | 시스템 운영정보 | 사용자, 역할, 기준정보 | 권한 변경·정정 관리 |

## 필수 서버 권한 테스트

- 제조 사용자가 검사결과를 수정할 수 없어야 함
- 품질 사용자가 권한 없는 프로젝트 번호로 API를 호출해도 데이터가 반환되지 않아야 함
- 물류 사용자가 품질 미승인 제품을 출발 처리할 수 없어야 함
- 영업 사용자는 허용 범위 밖 기사 개인정보를 볼 수 없어야 함
- UI에서 메뉴를 숨겨도 직접 API 호출은 거부되어야 함
