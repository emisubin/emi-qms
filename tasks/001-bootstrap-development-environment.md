# TASK-001: 개발환경과 저장소 골격 구성

## 목적

업무 기능을 구현하기 전에 로컬에서 반복 실행·테스트할 수 있는 프런트엔드, 백엔드, PostgreSQL 기반 개발환경을 구성합니다.

## 작업 범위

1. 저장소 문서와 ADR을 읽고 충돌 여부를 보고합니다.
2. 현재 지원되는 안정 버전을 조사해 다음을 구성합니다.
   - React + TypeScript 프런트엔드
   - ASP.NET Core Web API 백엔드
   - PostgreSQL 로컬 DB
3. 루트에서 전체 개발환경을 시작·종료할 수 있는 명령 또는 스크립트를 제공합니다.
4. 백엔드에 인증이 필요 없는 `/health/live`, `/health/ready` 엔드포인트를 추가합니다.
5. 프런트엔드에 API 상태를 표시하는 최소 개발용 시작화면을 추가합니다.
6. 환경변수 예시를 제공하고 실제 비밀값은 커밋하지 않습니다.
7. 프런트엔드 lint·typecheck·test, 백엔드 build·test 명령을 구성합니다.
8. GitHub Actions 또는 동등한 CI에서 빌드와 테스트가 실행되도록 합니다.
9. 실행법을 README에 반영합니다.

## 제약

- 아직 Entra ID, 실제 Azure, 실제 고객 데이터를 연결하지 않습니다.
- 업무 테이블과 화면을 임의로 대량 생성하지 않습니다.
- 마이크로서비스, Kubernetes, 메시지 브로커를 도입하지 않습니다.
- 버전과 핵심 라이브러리 선택 이유를 ADR 보충문서 또는 작업보고에 기록합니다.

## 완료 조건

- 새 개발자가 문서만 보고 로컬 실행 가능
- 프런트엔드에서 백엔드 상태 확인 가능
- PostgreSQL 준비상태가 health check에 반영됨
- 모든 자동 테스트 통과
- 비밀정보 검사에서 실제 비밀값 없음
- 변경 파일, 실행 명령, 테스트 결과, 남은 위험 보고
