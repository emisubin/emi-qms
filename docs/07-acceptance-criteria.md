# 7. 시스템 공통 인수 기준

## 기능

- 제품 QR 스캔 시 로그인 사용자의 역할에 맞는 기본 작업이 표시됩니다.
- PC에서 프로젝트별 모든 제품 상태를 조회할 수 있습니다.
- 모든 활성 사내 사용자는 전체 프로젝트와 부서별 기능 현황을 조회할 수 있습니다.
- 상태 전이가 업무 규칙에 맞지 않으면 서버가 거부합니다.
- 최초 검사, NCR, 조치, 재검사가 시간순으로 보존됩니다.
- 모든 품질 항목이 적합하고 열린 NCR이 없을 때만 품질 승인됩니다.
- 품질 승인 전 제품은 물류 출발 처리할 수 없습니다.
- 납품증빙과 영업 확인 전 제품은 종료되지 않습니다.

## 보안

- 로그인하지 않은 사용자는 업무 데이터를 볼 수 없습니다.
- `Project.Read.All`이 없는 제한 계정은 배정되지 않은 프로젝트·제품 API 요청 시 데이터 없이 거부됩니다.
- 전체 조회권한은 쓰기권한이 아니며 등록·수정·완료·취소 API는 담당 부서 Permission을 서버에서 검사합니다.
- 판매금액과 제조 소요시간은 Sales와 System Administrator에게만 반환됩니다.
- 프로젝트 삭제는 Sales만 가능하며 실제 SQL DELETE가 아니라 삭제 보관함으로 이동하는 논리삭제입니다.
- 삭제 보관함은 Sales와 System Administrator만 조회할 수 있습니다.
- 삭제된 프로젝트는 일반 목록과 일반 상세 API에서 제외되며 동일 Project Title 재사용이 가능합니다.
- 첨부파일 URL을 알아도 권한 없이는 다운로드할 수 없습니다.
- 비밀정보가 저장소, 응답, 로그에 노출되지 않습니다.

## 이력

- 등록·수정·승인·정정·상태변경·파일 다운로드의 주체와 시간이 기록됩니다.
- 승인 이후 변경에는 정정 사유와 승인자가 필요합니다.
- 과거 Revision과 과거 PDF는 최신 기준 변경으로 바뀌지 않습니다.

## 사용성

- 주요 모바일 입력은 현장에서 한 손 조작을 고려합니다.
- 필수 사진과 필수값은 완료 전 명확히 안내됩니다.
- 네트워크 오류 시 중복 저장을 방지하고 사용자에게 결과를 알립니다.
